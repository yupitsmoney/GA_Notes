Relations in forms and controllers

Learning Objectives

  • create a form for a relation

First steps

What we want to do is modify our app that we've been building (to run through this exercise again on your own, make a copy of the original coffeeshop folder which is found in the repo 04_week and start from there)

Right now we have beans but we need to add another model into our coffee app.

Say our client wants to be able to track where these different beans come from by Country and Province

Now lets create the new PlaceOfOrigin model, and lets assume that origin is based on the country and the province rails g model place_of_origin country province

What relation will our Bean have to PlaceOfOrigin

we need to add the line to our migration

rails g migration AddBeanIdToPlaceOfOrigins bean_id:integer

and the requisite lines to both of our models

class Bean < ActiveRecord::Base
    has_one :place_of_origin
end

class PlaceOfOrigin < ActiveRecord::Base
    belongs_to :bean
end

RESET

  • Because we're playing around with stuff right now we need to clear out anything thats in our db/seeds.rb file so we can start with a clean slate
  • Run rake db:reset to go back to a clean state DB wise
  • then open up your rails console to make sure our code works in rails console

Putting it into forms

Add this line 


accepts_nested_attributes_for :place_of_origin

to your Bean model this will let us create PlaceOfOrigin at the same time as we create our beans

and lets try it out in rails console rb bean = Bean.new(name: "Grant's Soylent Blend", roast: "Californian", quantity: 8, place_of_origin_attributes: { country: "USA", province: "Southern California"})

Does this create both the Bean and and the PlaceOfOrigin at once?

update our controller to use our new nested stuff

we need to create the new PlaceOfOrigin on our @bean in the controller so that way we can fill in the fields for it

# in the beans controller
  def new
    @bean = Bean.new
    @bean.place_of_origin = PlaceOfOrigin.new
  end
<h1>Add a new bean!</h1>

    <%= form_for @bean do |f| %>
      Name: <%= f.text_field :name %>
      Roast: <%= f.text_field :roast %>
      Origin: <%= f.fields_for :place_of_origin do |origin_form| %>
          <%= origin_form.label :province %>:<%= origin_form.text_field :province %><br>
          <%= origin_form.label :country %><%= origin_form.text_field :country %><br>
        <% end %>
      Quantity: <%= f.text_field :quantity %>
        <%= f.submit "Add beans!" %>
    <% end %>

Then we need to modify our create action in our controller to use these new params to create a bean

  #code below could be in private params or the one line (params.include... etc)could be included in create method
  
    def create
    @bean = Bean.new(params.require(:bean).permit(:name, :roast, :quantity, place_of_origin_attributes: [:province, :country]))
    if @bean.save and @bean.place_of_origin.save
      redirect_to beans_path
    else render :new
    end
  end

And make sure to include the @bean.place_of_origin.save in the create method

modify the show action to display our new attributes

<p>This coffee is a <%= @bean.roast %> roast from <%= @bean.place_of_origin.country %> in the province of <%= @bean.place_of_origin.province %>, and we have <%= @bean.quantity %> pounds available.</p>

Beans are supplied by vendors

Make sure to add this line to our bean show page

  <%= link_to "Add a Vendor", new_bean_vendor_path(@bean) %>

lets create a vendor model that has a name and city/state in which it is located and then generate the requisite controller as well Then we need to add a route for this

rails g model vendor name city state

check the migration to make sure it looks like this...

class CreateVendors < ActiveRecord::Migration
  def change
    create_table :vendors do |t|
      t.string :name
      t.string :city
      t.string :state

      t.timestamps null: false
    end
  end
end

  get "/beans/:bean_id/vendors/new" => "vendors#new", as: :new_bean_vendor

Why does our url have bean_id here and not just id?

Let's update our vendor model to accomodate that.

rails g migration AddBeanIdToVendors bean_id:integer

    class AddBeanIdToVendors < ActiveRecord::Migration
  def change
    add_column :vendors, :bean_id, :integer
  end
end

once that migration file has been created looks ok, run the migration in terminal

Look up the bean in the controller and also create a new vendor

class VendorsController < ApplicationController
  def new
    @bean = Bean.find(params[:bean_id])
    @vendor = Vendor.new()
  end
end

Our form should allow us to create the new vendor

<h1> New vendor </h1>

<%= form_for [@bean, @vendor]  do |f| %>
  <%= f.label :name %><%=f.text_field :name %>

  <%= f.label :city %><%=f.text_field :city %>
  <%= f.label :state %><%=f.text_field :state %>
  <%= f.submit "add a vendor"%>
<% end %>

Make sure that we generate the route that posts to the VendorsController create action, we have to make sure that the :bean_id is passed in though

The Vendor#create method

The #create method looks similar to ones we've seen so far the only difference is that we find the @bean that we are on first, and then we push to that bean's locations array

  def create
    @bean = Bean.find(params[:bean_id])
    @bean.vendors.push(Vendor.new(params.require(:vendor).permit(:name, :city, :state)))
    @bean.save
  end

now we can display our vendors inside of our bean show viewpage.

<h4>Vendors:</h4>
<% @bean.vendors.each do |vendor| %>
    <p><%= vendor.name %></p>
<% end %>

CODE TIP: you could refactor your vendor controller to add private params like we did with our beans... and it would look like this

class VendorsController < ApplicationController

    def new
        @bean = Bean.find(params[:bean_id])
        @vendor = Vendor.new()
    end

    def create
        @bean = Bean.find(params[:bean_id])
        
        # see new code here, accessing our private params
        @bean.vendors.push(Vendor.new(vendor_params))
        if @bean.save
            redirect_to beans_path
        else
            render "new"
        end
    end

    #and new code below
    private
    
    #made our params private
    def vendor_params
      params.require(:vendor).permit(:name, :city, :state)
    end
end
